# CodeX 作業ルール
## 基本方針
CodeXはIssueとdocs配下の要件を参照し、最小変更で実装すること。
## 参照順序
1. GitHub Issue
2. docs/requirements/hello-world.md
3. README.md
4. 既存コード
## 作業ルール
- 要件にない機能は追加しない
- 既存構成を大きく変更しない
- 不要なライブラリを追加しない
- 大規模なリファクタリングをしない
- 変更理由をPull Request本文に記載する
- 確認方法をPull Request本文に記載する
## 禁止事項
- 認証機能を追加しない
- DBを追加しない
- API通信を追加しない
- DIライブラリを追加しない
- Clean Architecture化しない
- 画面を複数作らない
